User Story 1

I can see a main element with a corresponding id="main-doc", which contains the page's main content (technical documentation).

Solution

<main id="main-doc">

User Story 2

Within the #main-doc element, I can see several section elements, each with a class of main-section. There should be a minimum of 5.

Solution

<section class="main-section" id="User_Story_2">

User Story 3

The first element within each .main-section should be a header element which contains text that describes the topic of that section.

Solution

<header> <h1>User Story 3</h1> </header>

User Story 4

Each section element with the class of main-section should also have an id that corresponds with the text of each header contained within it. Any spaces should be replaced with underscores (e.g. The section that contains the header "Javascript and Java" should have a corresponding id="Javascript_and_Java").

Solution

<section class="main-section" id="User_Story_4">

User Story 5

The .main-section elements should contain at least 10 p elements total (not each).

Solution

<p>This tage should appear at least 10 different times.</p>

User Story 6

The .main-section elements should contain at least 5 code elements total (not each).

Solution

Choose whichever 5 tags you wish to use anywhere in your page

User Story 7

The .main-section elements should contain at least 5 li items total (not each).

Solution

User Story 14

On regular sized devices (laptops, desktops), the element with id="navbar" should be shown on the left side of the screen and should always be visible to the user.

Solution

Your CSS should include the following for the #id navbar
